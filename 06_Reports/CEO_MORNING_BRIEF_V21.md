## 0. 基本思想（v2.1で追加される原則）

- Morning Briefは「会社の情報を全部報告するレポート」ではない。目的はただ一つ: **CEOが『今日、何を決めれば会社が一番前に進むか』を5分で判断できること**
- Morning Briefは情報を増やすものではなく、**AIが情報を整理し、CEOの認知負荷を減らす**ものである
- AIは**重要ではない情報を隠す勇気**を持つ（隠す≠捨てる。§4のとおり全情報はDashboard・indexに保持され、いつでも見られる）
- Morning BriefはDashboardの要約ではない。**詳細はDashboardに任せ、Briefは判断だけを扱う**

## 1. 最重要ルール

| ルール | 内容 |
|---|---|
| **7項目上限** | CEOが朝見る情報は**最大でも7項目程度**（§2の情報ブロック数で数える） |
| **スクロールなし** | スクロールしなくても読める量を目標（目安: 30行以内） |
| **判断は原則1件** | 多くても3件。判断候補が20件あってもAIが整理して絞る |
| **評価基準は一つ** | 情報量では評価しない。**「CEOが5分以内で今日の経営判断を終えられるか」が唯一の評価基準** |

## 2. v2.1の構成（v2.0の⓪+7セクションを5ブロックへ圧縮）

```
💬 AIから一言        最重要要素（今日会社を一番前に進める判断を一言で）
① 今日の判断         原則1件・最大3件
② AIからの提案       FOS Review結果・最大3件に厳選（Brief最大の価値）
③ AI Actions         「②お願い」だけで実行（番号付き・最大5件）
④ 会社の状態         AIの1〜3行要約（数字の羅列禁止・詳細はDashboardへ）
（条件付き表示）      催事があった日のみ催事1行 / 結果確認期限が来た日のみResult 1件
```

### 💬 AIから一言（Morning Briefで最も重要な要素）

「今日会社を一番前に進める判断」を一言で伝える。型:

```
おはようございます。
今日はso uの発注上限を決めれば会社は止まりません。
スタッフが回答を待っています。
```

- 3〜5行。**挨拶 → 今日の中心判断（①の先頭と必ず一致）→ その理由を一言（誰が待っているか・何が動くか）**
- 異常・催事・同期は**特記があるときだけ**1行追加（「異常なし」が常態になったら書かない。沈黙=正常をBriefでは採用し、正常の明示はDashboardが担う）
- 事実のみ・演出なし（静けさと余白・憲法第6条）

### ① 今日の判断（原則1件）

- **1件が原則**。Sランク複数・期限切れ等の場合のみ最大3件
- 絞る順: S → 期限切れ → スタッフ待ち → 契約 → お金 → 発注（v2.0 §2-③継承）。選外は表示しない（⛔「今日やらないこと」リストはBriefから削除し、判断ログ側に保持）
- 各判断は**5行以内**: 判断1行 / 推奨1行 / 理由・リスク1〜2行 / 記入欄1行（5点セット全文はリンク先=Draftへ）

### ② AIからの提案（FOS Review・Brief最大の価値）

- AIがBrief生成前（夜間）にFOS全件をレビューし、**最大3件に厳選して**提案する
- 検出6観点（v2.0継承）: 完了忘れ / 重複 / 順番がおかしい / AIでできる / 待ち状態 / 今日やらなくても良い
- 表示は「1提案=1行+根拠」。4件以上検出した場合は影響の大きい順に3件、残りは翌日以降へ回す（毎日3件ずつ会社が整っていく設計）
- **AIはFOSを変更しない。提案だけ**（不変）

### ③ AI Actions（重要項目）

- AIだけで終わる仕事を番号付きで提示し、CEOは**「②お願い」だけで実行できる**状態を目指す
- 最大5件。前提材料が必要なものは1語で明示（例:「要メモ共有」）
- 実行範囲はDraft/取込/生成/整理まで（対外送信・支払・発注はCEOのみ・不変）

### ④ 会社の状態（数字を並べない）

- **AIが要約する**。型: 「**会社は正常です。注意点は2件あります** — ①オージェイズ未入金（期限未設定）②Shopify等6ソース未接続」
- 数値の羅列・全項目リスト・未接続一覧は書かない。**変化・異常・注意だけ**を1〜3行
- 詳細（売上数値・Dataset状態・Health・Learning件数）は**すべてDashboardへ**（役割分担維持: Dashboard=会社の現在地を見る画面 / Brief=CEOが今日判断する画面。**Dashboardは変更しない**）

### 条件付き表示（出す条件があるときだけ現れる）

| ブロック | 出す条件 | 出さない日 |
|---|---|---|
| 催事 | 昨日〜今日に催事があった時のみ（売上・日商・TOP商品を各1行） | 非表示（「催事なし」の行も書かない。一言で触れるのは特記時のみ） |
| Result確認 | review_after_days経過など確認期限が来た時のみ1件ずつ | 非表示 |
| 緊急アラート | 発生時のみ最上部（v1.0 §6継承） | 非表示 |

## 3. v2.0からの削除・移動（隠す勇気の適用）

| v2.0の要素 | v2.1での扱い |
|---|---|
| ⓪昨夜のAI作業 | **Briefから削除** → 一言に吸収（特記時のみ）。作業ログはDashboard/AI Systemへ |
| ①Company Status（数値表） | **要約1〜3行へ圧縮**。数値はDashboardへ |
| ②Event Status（毎日表示） | **条件付き表示へ**（催事があった日だけ。「昨日は催事はありませんでした」の定型行も廃止=書かないことが「なし」を意味する） |
| ⛔今日やらないこと（選外リスト） | **Briefから削除**。選外理由はDraftログに保持（消えていないことはログが保証） |
| 📋レビュー待ち | **Briefから削除** → Dashboardへ |
| ⏭次に決めること | **Briefから削除** → 週次レビューへ移管（日次は今日だけを扱う） |
| ⑥Result Review（毎日表示） | **条件付き表示へ**（確認期限が来た日だけ） |
| ⑦AI System | **Briefから完全削除** → Dashboardへ（AI開発情報はCEOの朝から退場） |

**残した情報**: AIから一言 / 今日の判断（1件原則）/ FOS Review提案（≤3件）/ AI Actions（≤5件）/ 会社の状態の要約 / 条件付き（催事・Result期限・緊急）。

## 4. 隠す勇気の保証（情報は消えない）

- Briefに出さない情報は、Dashboard（現在地）・index.json（Dataset）・decision_log_draft（選外理由）・PENDING（AI開発）が**全件保持**する
- CEOが「詳しく」「全部見せて」と言えば、AIはいつでも該当詳細を出す（隠すのは表示であり、記録ではない）
- 非表示判定はルールベース（§2の条件）のみ。**AIの恣意的な省略はしない**（迷ったら出すのではなく、迷ったらルールに従う）

## 5. 評価基準（唯一）

**CEOが5分以内で今日の経営判断を終えられるか。** 補助指標: Brief 30行以内 / 判断記入完了までの時間 / 「詳しく」と聞き返された回数（多ければ隠しすぎ・翌日調整）。情報量・網羅性では評価しない。

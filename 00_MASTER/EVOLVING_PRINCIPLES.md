# EVOLVING_PRINCIPLES — 成長する経営判断基準

> 本書は**現在有効な会社の判断原則**を管理する。ただしCORE_PRINCIPLESではない。
> ここに載る原則（EP）は実際の経営判断で継続運用され、複数ブランド・複数事業で有効性が確認され、**CEOが承認した場合のみ**COREへ昇格する。AIは勝手に昇格させない。

- 文書ID: MASTER-07（読込順7）
- 版: v0.2
- 最終更新: 2026-07-06
- 上位文書: [CORE_PRINCIPLES.md](./CORE_PRINCIPLES.md)
- 書込権限: AIはCEO承認済みPrincipleの登録・運用記録の追記のみ可（昇格・削除はCEOのみ）

---

## 1. 学習サイクル（Architecture v1.3）

```
Conversation → Insight / Decision → Pattern → Lesson → Principle
→ CEO Review → EVOLVING_PRINCIPLES（本書・active運用）
→ CORE_PRINCIPLES（CEOのみ昇格可）
```

## 2. 運用ルール

1. 登録できるのは**CEOレビューでApproveされたPrinciple**のみ（KNOWLEDGE_PROMOTION_RULES準拠）
2. EPは「現在有効」な判断原則として、AI・Agentの判断時に参照される（COREと矛盾する場合はCOREを優先）
3. CORE昇格の条件: ①実際の経営判断での継続運用 ②複数ブランド・複数事業での有効性確認 ③CEO承認。**AIは昇格を提案できるが、実行はCEOのみ**
4. 運用実績（EPを参照した判断・結果）は各EPの運用記録に追記していく

### EP-001: ブランドは売るものではなく、人の歩みを豊かにするために存在する

- **本文**: すべてのブランド・商品・発信は「歩みゆたかに」を最上位思想とし、作り手の情熱とストーリーを大切に形にする。売ることではなく、人の歩みを豊かにすることを目的とする。
- **根拠Principle**: PRN-001 / **根拠Lesson**: LSN-001 / **根拠Pattern**: PTN-001（6会話反復・2024-08〜2026-04）
- **採用日**: 2026-07-06 / **Status**: active / **Version**: v1.0
- **CORE昇格候補**: ◎（CEO「会社・ブランド・AIすべての最上位思想」。憲法第1・2条と完全整合）
- **CORE昇格条件**: 全ブランドの発信・商品判断での継続運用を確認のうえCEO承認。既存CORE第1〜3条との統合方法もCEOが決定
- **運用記録**: （未記入）

### EP-002: お客様の想いは、必ず作り手まで届ける

- **本文**: お客様から預かった想い（手紙・言葉・背景)は販売側で止めず、必ず職人・作り手まで共有する。想いが届いた上でのものづくりが、最速・最良の納品につながる。
- **根拠Principle**: PRN-002 / **根拠Lesson**: LSN-002 / **根拠Pattern**: PTN-002（5会話反復・2023-06〜2026-07）
- **採用日**: 2026-07-06 / **Status**: active / **Version**: v1.0
- **CORE昇格候補**: ○（so u標準対応として承認済み）
- **CORE昇格条件**: so u以外のブランド（SUNNY NOMADO等の職人連携）でも同運用が確認され、CEOが承認したとき
- **運用記録**: （未記入）

### EP-003: 一人のお客様のために、通常を超える対応を惜しまない

- **本文**: 目の前の一人に最上を尽くす。必要であれば通常は行わない特別な製作体制を整える。数ではなく一人に向かうことが、ブランドの信頼をつくる。
- **根拠Principle**: PRN-003 / **根拠Lesson**: LSN-005 / **根拠Pattern**: PTN-004（3会話反復）
- **採用日**: 2026-07-06 / **Status**: active / **Version**: v1.0
- **CORE昇格候補**: ○（憲法第3条「迎える」の実践形）
- **CORE昇格条件**: 複数ブランドでの特別対応事例の蓄積とCEO承認
- **運用記録**: （未記入）

### EP-004: 安くするのではなく、工夫によってお客様へ還元する

- **本文**: 値引きで選ばれることを良しとしない。DX・改善・工夫でコストを下げ、その分をお客様へ還元する。お客様のためを考え抜いた結果として会社が残る。
- **根拠Principle**: PRN-004 / **根拠Lesson**: LSN-016, LSN-004 / **根拠Pattern**: PTN-003
- **採用日**: 2026-07-06 / **Status**: active / **Version**: v1.0
- **CORE昇格候補**: ◎（CORE第4条「安売りしない」の実践形として統合候補）
- **CORE昇格条件**: 値付け・販促の実判断での継続運用確認後、CORE第4条への追記としてCEO承認
- **運用記録**: （未記入）

### EP-005: 判断は過去実績の数値を基準に持ちつつ、今年の条件・環境・在庫・市場変化を含めて行う

- **本文**: 実績数値（例: 駅系催事の平均売上30万）は判断の土台として必ず参照する。ただし前年実績だけで判断せず、今年の条件・環境・在庫・市場変化を重ねて最終判断する。
- **根拠Principle**: PRN-005 / **根拠Lesson**: LSN-003 / **根拠Pattern**: —
- **採用日**: 2026-07-06 / **Status**: active / **Version**: v1.0
- **CORE昇格候補**: ○（催事AI・発注在庫AIの中核判断原則）
- **CORE昇格条件**: 催事以外（発注・広告等）の判断でも有効性が確認され、CEOが承認したとき
- **運用記録**: （未記入）

### EP-006: 失敗は謝罪で終わらせず、プロセスの見直しへ昇華させる

- **本文**: 事故・失敗が起きたら、個別の謝罪・対処で終わらせず、必ず事前準備プロセス・チェックリストの見直しへつなげる。仕組みでの再発防止だけが「継続」を守る。
- **根拠Principle**: PRN-006 / **根拠Lesson**: LSN-023 / **根拠Pattern**: —
- **採用日**: 2026-07-06 / **Status**: active / **Version**: v1.0
- **CORE昇格候補**: ○（CEO「継続するために、未然の防止」）
- **CORE昇格条件**: 催事以外の失敗対応でも運用され、CEOが承認したとき
- **運用記録**: （未記入）

### EP-007: 品質は時間と反復で確保し、確保できるまで世に出さない・価格を約束しない

- **本文**: 試作と修正の反復に時間を惜しまない。自信を持てる品質に達してから世に出す。品質が確定する前に価格を約束しない。
- **根拠Principle**: PRN-007 / **根拠Lesson**: LSN-018, LSN-014 / **根拠Pattern**: —

- **CORE昇格条件**: 商品企画・ODM両方での継続運用とCEO承認
